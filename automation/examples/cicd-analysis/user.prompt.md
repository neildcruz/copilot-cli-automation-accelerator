# CI/CD Pipeline Analysis User Prompt

Analyze CI/CD pipeline configurations and workflows:

1. Review all pipeline files (.github/workflows, .gitlab-ci.yml, azure-pipelines.yml, etc.)
2. Identify bottlenecks and optimization opportunities
3. Check for security best practices (secrets, permissions, supply chain)
4. Validate test coverage and quality gates
5. Review deployment strategies and environment configurations
6. Assess monitoring and alerting setup
7. Check for proper error handling and notifications
8. Suggest improvements for build time and reliability

Generate a comprehensive report with prioritized recommendations.
